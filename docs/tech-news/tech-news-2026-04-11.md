- **出典**: Publickey
- **注目ポイント**: S3のユースケースを劇的に拡大する画期的な機能追加
- **詳細**: AWSがAmazon S3バケットをNFSファイルシステムとしてアクセスできる「Amazon S3 Files」の提供を開始した。Amazon EFSをベースにファイルシステム操作をS3リクエストに変換する仕組みで、これまでオブジェクトストレージとして利用するためにアプリケーション側で専用のAPIを使う必要があったS3が、通常のファイルシステムとしてマウントして利用可能になった。リードスループットは最大で毎秒数テラバイト、レイテンシはミリ秒単位という高性能を実現しており、数千のNFSクライアントからの同時アクセスにも対応する。既存のS3バケットのデータはそのまま利用でき、書き込みデータはまず高耐久性の高速ストレージに保存された後にS3バケットに同期される。この機能により、従来ファイルシステムインターフェースが必要だったレガシーアプリケーションとS3の統合が容易になり、インフラアーキテクチャの簡素化が期待される。Publickeyの人気記事ランキングでも1位を獲得している。

### [Summary of CVE-2026-23869 - Vercel](https://vercel.com/changelog/summary-of-cve-2026-23869)

- **出典**: Vercel
- **注目ポイント**: Next.jsを利用する全プロジェクトに影響する高深刻度の脆弱性
- **詳細**: React Server Componentsに高深刻度のDoS脆弱性（CVSS 7.5）が発見された。App Routerを使用するServer Functionエンドポイントに特殊なHTTPリクエストを送信すると、デシリアライズ時に過剰なCPU使用が発生し、サービス拒否攻撃が成立する。Next.js 13.x、14.x、15.x、16.xの全バージョンが影響を受ける。Vercelは自社WAFに対策ルールを展開し全プロジェクトを無償で保護しているが、WAFだけでは完全な保護にならないため、Next.js 15.5.15または16.2.3へのアップグレードが強く推奨される。Next.jsを使用しているプロジェクトでは速やかなバージョン確認と対応が求められる。
